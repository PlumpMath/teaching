µC → µC

Try and get two Arduinos to communicated using I2C, Serial, or some other protocol. Once you have that working, design and implement a protocol of your own for the Arduinos to communicate with. Is there any reason to use something if your own design? How does it compare with I2C or Serial?

Final Project

Your final project is to design and implement a system that meets the definition of a Large System we have been using: a system where code running in more than one place must coordinate. I strongly encourage overlapping this project with projects in other classes if possible.

Send me your proposal if you have not already. In addition to a proposal, prepare a network diagram of the kind I have been drawing in class. You do not have to be very formal about it, but make sure to include:

  • Every node in your system
    • What kind of machine is it? Is it a laptop? An arduino? Is there anything particular about it?
  • Arrows between nodes to indicateo the flow of information
    • What direction is the information flowing?
    • What protocol is being used?
    • How is the data formatted?
